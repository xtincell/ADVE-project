## Notable patterns shipped this session (reusable downstream)

1. **Pure-function-decoupled-from-IO** (Story 4.2 `scoreFromActions` vs `runAttribution`) — the pure core takes plain TS objects so unit tests need no Prisma mocks. Reuse for any measurement that wants synthetic-data testability.
2. **Discriminated-union return + typed-cause alphabet** (Stories 4.1/4.3 `AttributionResult` / `CohortRetentionMeasurement` + `SuperfanInsufficientReason`) — the P22-2 pattern. Every measurement handler returns `OK | INSUFFICIENT_DATA(+typed reason)`, never a nullable score.
3. **Exhaustive ConnectorResult switch with defensive-returns-not-throw** (Story 4.3) — façade pattern : `switch (signal.state) { LIVE / DEFERRED / DEGRADED }`, no `default`, no throw across the boundary ; defensive cases (NOT_FOUND, TENANT_MISMATCH) return INSUFFICIENT_DATA branches.
4. **Tolerant vocabulary mapper** (Story 4.4 `extractLineage`) — when the repo carries multiple enum vocabularies for the same concept, a `switch`-based normalizer onto the canonical alphabet (mapping, not reinvention).
5. **Schema-equals-runtime-shape via refine** (Story 4.5 `attributionCoefficientsSchema`) — `z.record(z.string(), z.number()).refine(keys ⊆ ALPHABET)` keeps the validator pinned to the runtime `Record<string, number>` shape instead of a parallel `z.object` that drifts.
6. **Manual-first parity = identical OK shape** (Story 4.5) — the structural test for ADR-0060 : assert both the LLM/algorithmic path and the manual path return the same result key set ; only an internal `mode` field discriminates.
