## 5. Design tokens — the 4-tier cascade

Read [DESIGN-TOKEN-MAP.md](../governance/DESIGN-TOKEN-MAP.md). Rules:

```
Tier 0 Reference (--ref-*)  →  Tier 1 System (--color-*)  →  Tier 2 Component (--button-*, --card-*, ...)  →  Tier 3 Domain (--pillar-*, --division-*, --tier-*, --classification-*)
```

**Three absolute interdits** (enforced by tests + lint):

1. No component in `src/components/**` consumes a Reference token directly (`var(--ref-*)`). Always System / Component / Domain. → Test: [tests/unit/governance/design-tokens-cascade.test.ts](../../tests/unit/governance/design-tokens-cascade.test.ts).
2. No raw Tailwind color class (`text-zinc-500`, `bg-violet-500`, `border-emerald-700`) outside `src/components/primitives/**` + `src/styles/**`. → Codemod `npm run codemod:zinc` + ESLint `lafusee/design-token-only` + test `design-tokens-canonical.test.ts`.
3. No inline `.join(" ")` or ternary variants when ≥1 variant exists. CVA is mandatory. → Test `design-primitives-cva.test.ts`.

## 7. Adding a new component — checklist

1. **Primitive vs. composed** — does this belong under `primitives/` or in a deck-specific folder?
2. **CVA** — if it has variants, use `class-variance-authority`.
3. **Token-only** — only System / Component / Domain tokens. No raw Tailwind colors.
4. **Manifest** — co-locate `<component>.manifest.ts` declaring variants, mission, a11y.
5. **Story** — add `<component>.stories.tsx` for Storybook.
6. **a11y** — keyboard navigation, ARIA roles, contrast AA.
7. **Test** — at minimum an interaction test or a Chromatic snapshot.
8. **Regenerate** — `npm run ds:components-map` to update [COMPONENT-MAP.md](../governance/COMPONENT-MAP.md).
