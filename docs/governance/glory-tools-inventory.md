> **Classification brief-only vs brief-to-forge** ([PANTHEON.md](PANTHEON.md), [ADR-0009](adr/0009-neter-ptah-forge.md)) : les Glory tools sont gouvernés par **Artemis** dans le sous-système Propulsion. À partir de Phase 9, chaque manifest Glory tool déclare un champ optionnel `forgeOutput?: ForgeSpec` :
>
> - **brief-only** (sans `forgeOutput`) : output texte consommé tel quel (ex: `brand-bible-extractor`, `long-copy-craftsman`, `dialogue-writer`). Reste intégralement Artemis.
> - **brief-to-forge** (avec `forgeOutput`) : produit un `ForgeBrief` avec `forgeSpec` qui handoff downstream à **Ptah** pour matérialisation (ex: `kv-prompt`, `script-writer` quand utilisé en pipeline vidéo, `jingle-brief`).
>
> La cascade de production complète : `Mestor → Artemis Glory tool → ForgeBrief avec forgeSpec → Ptah materialize → AssetVersion`.
>
> Chaque tool brief-to-forge déclare aussi un `manipulationProfile?: ManipulationMode[]` (modes compatibles parmi peddler/dealer/facilitator/entertainer — voir [MANIPULATION-MATRIX.md](MANIPULATION-MATRIX.md)). Mestor pre-flight `MANIPULATION_COHERENCE` gate refuse les invocations qui sortent du `Strategy.manipulationMix`.
