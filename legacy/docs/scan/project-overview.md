## 1. What this is

**La Fusée** is an **Industry OS** for the African creative market, built and operated by the **UPgraders** agency. It is **not** a SaaS product, not a "platform" — it is the operating system that runs an entire creative-industry value chain (brief intake → strategy → production → talent dispatch → comms → invoicing) end-to-end.

**Mission (north star)** — transform brands into cultural icons by industrializing the accumulation of *superfans* who shift the *Overton window* in their sector. Full statement: [docs/governance/MISSION.md](../governance/MISSION.md).

The codebase is the OS itself. Every module exists to serve that mission; modules that don't are flagged as drift (see [`audit-mission-drift.ts`](../../scripts/audit-mission-drift.ts) and the `mission-drift.yml` workflow).

## 2. Three plans you must keep separate

| Plan | What it is | Example |
|---|---|---|
| **Portail (UI)** | Where users live | Console (UPgraders/operators), Cockpit (founders), Agency, Creator, Intake (public) |
| **Livrable (output)** | What gets produced | `BrandAsset.kind` — `BIG_IDEA`, `CREATIVE_BRIEF`, `MANIFESTO`, `ORACLE_DOCUMENT`, claim, KV, etc. |
| **OS (engine)** | La Fusée itself | Invisible to clients — Mestor / Artemis / Ptah / Seshat / Thot / Imhotep / Anubis governance cascade |

Confusing these three is the most common source of architectural drift.

## 5. Architecture (one-screen view)

**Strict 7-layer hierarchy** — see [architecture.md](./architecture.md) for the full picture.

```
Layer 0  src/domain/                pure types, enums, branded primitives, Zod schemas
Layer 1  src/lib/                   utilities, db client, auth helpers, design tokens
Layer 2  src/server/governance/     manifests, registry, event bus, Mestor dispatcher, NSP server, hash-chain, tenant-scoped DB, governed-procedure
Layer 3  src/server/services/       business services (100 dirs) — Artemis, Ptah, Seshat, Thot, Imhotep, Anubis, LLM gateway, Glory tools, Oracle...
Layer 4  src/server/trpc/           100 routers, protected by governedProcedure / auditedProcedure
Layer 5  src/components/neteru/     Neteru UI kit (MestorPlan, ArtemisExecutor, SeshatTimeline, OracleEnrichmentTracker)
Layer 6  src/app/, src/components/*  pages + ad-hoc UI
```

Layer N may import ≤ N. Enforced by `eslint-plugin-boundaries` ([eslint.config.mjs](../../eslint.config.mjs)) + `madge --circular`.

**Governance cascade** — every business mutation must traverse `mestor.emitIntent()`:

```
client/operator/cron → Mestor → Thot (budget gate) → Seshat (read context)
                              → Artemis (dispatch brief Glory tool) → Ptah (materialize asset)
                              → Bus (hash-chained IntentEmission + NSP SSE replay)
```

7 active Neteru (cap APOGEE 7/7 since Phase 14/15): **Mestor** (guidance) · **Artemis** (briefs) · **Ptah** (forge) · **Seshat** (telemetry + Tarsis) · **Thot** (fuel/budget) · **Imhotep** (crew programs) · **Anubis** (comms + ad networks + credentials vault). NEFER is the **operator** (this agent), not a Neter.

Full panthéon narrative: [PANTHEON.md](../governance/PANTHEON.md). Framework axioms: [APOGEE.md](../governance/APOGEE.md). Layering rationale: [ADR-0002](../governance/adr/0002-layering-six-couches.md).
