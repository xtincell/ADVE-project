## Descriptions

- **ACTIVATE_RETAINER** — Activate a retainer subscription tier (BASE / PRO / ENTERPRISE) for an operator/strategy.
- **ADMIN_SET_USER_ROLE** — Console superviseur (Vague 7) : promotion/retrogradation du role d'un compte (entrepreneur/createur/agence/partenaire). Trace + audit.
- **ANUBIS_BROADCAST_MESSAGE** — Lance un broadcast multi-canal pour un CommsPlan. Crée un BroadcastJob queued pour fan-out async. Les providers façades retournent DEFERRED_AWAITING_CREDENTIALS si pas de creds (cf. ADR-0021).
- **ANUBIS_BUY_AD_INVENTORY** — Achète de l'inventaire ad sur Meta/Google/X/TikTok via le provider sélectionné. Retourne DEFERRED si credentials absentes (Credentials Center à configurer).
- **ANUBIS_CANCEL_BROADCAST** — Annule un BroadcastJob queued ou scheduled (status → CANCELLED). No-op si déjà SENT.
- **ANUBIS_DRAFT_COMMS_PLAN** — Draft un plan comms (audience cible, canaux, timing) pour une stratégie/campagne. Phase 15+ status=DRAFT (vrai draft) ; back-compat Phase 13 dormant si strategy absente. Cf. ADR-0011 + ADR-0020.
- **ANUBIS_FETCH_DELIVERY_REPORT** — Pull metrics du provider externe (delivered, bounced, opened) et update DeliveryReceipt. Retourne DEFERRED si credentials absentes.
- **ANUBIS_MCP_INVOKE_TOOL** — Invoque un tool MCP entrant (Slack/Notion/Drive/Calendar/Figma/GitHub) via McpRegistry direction=INBOUND. Retourne DEFERRED si credentials absentes (cf. ADR-0021/0022).
- **ANUBIS_MCP_REGISTER_SERVER** — Enregistre un McpRegistry (INBOUND ou OUTBOUND) avec credentialRef vers ExternalConnector. Cf. ADR-0026.
- **ANUBIS_MCP_SYNC_REGISTRY** — Découvre la liste des tools disponibles d'un MCP server inbound et met à jour McpRegistry.toolsCache.
- **ANUBIS_OAUTH_DEVICE_FLOW_POLL** — Poll OAuth token endpoint pour récupérer access_token+refresh_token quand le user a autorisé. Persiste dans Credentials Vault si OK. Cf. ADR-0048.
- **ANUBIS_OAUTH_DEVICE_FLOW_START** — Démarre OAuth 2.1 device flow (RFC 8628) pour un MCP server externe. Discovery RFC 9728 + POST device_authorization_endpoint. Retourne verification_uri_complete à présenter au user. Cf. ADR-0048.
- **ANUBIS_OAUTH_REFRESH_TOKEN** — Refresh manuel d'un OAuth access_token via refresh_token. Auto-déclenché par mcp-client si expires_at < now+60s. Cf. ADR-0048.
- **ANUBIS_PUSH_NOTIFICATION** — Push une notification unifiée (in-app via Notification model + NSP SSE realtime + Web Push fan-out + email/SMS optionnels). Respecte NotificationPreference (channels, quiet hours, digestFrequency).
- **ANUBIS_REGISTER_CREDENTIAL** — Enregistre/update un ExternalConnector (API key, OAuth tokens, account ID) pour un provider externe via Credentials Center UI. Cf. ADR-0021.
- **ANUBIS_REGISTER_PUSH_SUBSCRIPTION** — Enregistre un PushSubscription Web Push (endpoint, p256dh, auth) pour un user. Idempotent par endpoint.
- **ANUBIS_RENDER_TEMPLATE** — Rend un NotificationTemplate (Handlebars + MJML) avec vars typées. Retourne { subject?, html?, text }.
- **ANUBIS_REVOKE_CREDENTIAL** — Soft-delete un ExternalConnector (status → INACTIVE). Conservé pour traçabilité audit.
- **ANUBIS_RUN_DIGEST** — Cron digest daily/weekly : groupe les notifications IN_APP non-lues par user et envoie un email de récap via template `notification-digest`.
- **ANUBIS_SCHEDULE_BROADCAST** — Planifie un broadcast pour un timestamp futur (BroadcastJob.scheduledFor). Lance via cron interne ou scheduler externe.
- **ANUBIS_SEGMENT_AUDIENCE** — Segmente une audience selon des règles JSON (sector, devotion tier, geo, etc.). Retourne segment + count estimé.
- **ANUBIS_TEST_CHANNEL** — Test connectivity d'un canal (envoi d'un ping de validation au provider). Marque ExternalConnector ACTIVE si succès.
- **ANUBIS_TRACK_DELIVERY** — Track delivery d'un BroadcastJob (delivered, opened, clicked) via DeliveryReceipt. Pull ou webhook selon provider.

- **APPLY_TO_MISSION** — Vague 7 : un talent/agence candidate a une mission ouverte (statut PENDING, message + taux propose).
